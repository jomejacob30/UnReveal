# Shadow DOM Unblur — Chrome Extension (Manifest V3)

## What It Does
Reveals client-side data that has been visually hidden using CSS filters, pseudo-element overlays, redacted fonts, or `-webkit-text-security` — even when the blur is applied inside Shadow DOM trees.

## Architecture

This extension uses a **dual-script strategy** to handle both open and closed Shadow DOMs:

### `intercept.js` (MAIN world, `document_start`)
- Runs **before** the page's own JavaScript
- Overrides `Element.prototype.attachShadow` to capture references to every shadow root — including **closed** ones
- Stores captured roots on a non-enumerable property of each host element
- Dispatches a custom DOM event so the content script can react in real time

### `content.js` (ISOLATED world, `document_idle`)
- Recursively walks the entire DOM tree to discover shadow hosts
- Injects targeted CSS into every shadow root to defeat:
  - `filter: blur()` and `backdrop-filter: blur()`
  - Pseudo-element (`::before` / `::after`) overlays
  - Redacted / obfuscation web fonts
  - `-webkit-text-security` disc rendering
  - `mask-image` and `clip-path` hiding
- Attaches `MutationObserver` **inside each shadow root** (not just `document.body`) to catch React/Vue/framework re-renders
- Listens for custom events from `intercept.js` for immediate processing of new shadow roots
- Runs follow-up scans at 1.5s and 4s to catch lazy-loaded components

## Installation

1. Open Chrome and navigate to `chrome://extensions/`
2. Enable **Developer mode** (toggle in the top-right corner)
3. Click **Load unpacked**
4. Select the `shadow-unblur-extension` folder
5. The extension icon should appear in your toolbar
6. Navigate to the target site — the unblur runs automatically

## Customization

### Restricting to a specific domain
In `manifest.json`, change the `"matches"` arrays from:
```json
"matches": ["<all_urls>"]
```
to your target domain:
```json
"matches": ["https://example.com/*"]
```

### Adding new selectors
If the site updates its class names, add new selectors to the `UNBLUR_CSS` constant in `content.js`. The hashed class names (like `fc6omth`) are generated by CSS-in-JS and may change between deployments.

## Limitations

1. **Server-side content gating**: If the actual text is never sent to the browser (replaced with placeholder text at the API level), this extension cannot recover it.
2. **Canvas-rendered text**: If content is drawn onto a `<canvas>` element rather than existing as DOM text nodes, CSS overrides have no effect.
3. **Closed shadow roots created before interception**: If a shadow root is attached during inline `<script>` execution that runs before `intercept.js` loads, it may be missed. This is extremely rare with Manifest V3's `document_start` timing.

## Debugging

Open DevTools Console and look for:
- `[Unblur] attachShadow interceptor installed` — confirms intercept.js loaded
- `[Unblur] Content script initialized — full scan complete` — confirms content.js ran

If the blur persists, run this in the console while inspecting the blurred element:
```js
console.log('Font:', getComputedStyle($0).fontFamily);
console.log('::before filter:', getComputedStyle($0, '::before').filter);
console.log('::after filter:', getComputedStyle($0, '::after').filter);
console.log('text-security:', getComputedStyle($0).webkitTextSecurity);
```
This will identify which technique is being used so the CSS payload can be adjusted.
